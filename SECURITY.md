# 安全策略

粉碎一切是不可逆数据销毁工具，安全问题的处理优先级高于普通功能请求。

## 支持版本

当前处于 `0.x` 快速迭代阶段。安全修复默认只面向最新 `main` 分支和最新发布版本。

## 报告漏洞

请优先使用 GitHub 的私有漏洞报告能力（Security Advisories / Private vulnerability reporting）。如果仓库尚未启用该能力，请先通过维护者公开联系方式建立私有沟通，不要在公开 issue 中直接贴可复现的破坏性 PoC。

报告时请尽量包含：

- 影响版本、Windows 版本、文件系统类型（NTFS/ReFS/FAT/exFAT）；
- 最小复现步骤；
- 预期行为与实际行为；
- 是否涉及误删、越权、路径护栏绕过、数据残留、日志泄露或崩溃；
- 可安全公开的日志片段。默认不要提交真实敏感路径或文件名。

## 安全边界

本项目承诺：

- 不后台联网；
- 日志默认脱敏路径；
- 系统目录、驱动器根目录等高风险路径硬拒绝；
- 符号链接、Junction、ADS、小文件 MFT 驻留等 Windows 文件系统边界纳入测试和审计。

本项目不承诺：

- 对 SSD/NVMe 闪存介质完成物理级覆写保证；
- 清除云盘、同步盘、备份软件、卷影复制、文件历史记录、远程副本中的数据；
- 在恶意内核驱动、受损系统或硬件级取证场景下提供保证。
