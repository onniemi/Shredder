# 发布检查清单

每次准备公开 Release 前按此清单走一遍。

## 构建

- [ ] `dotnet restore src/Shredder.sln`
- [ ] `dotnet build src/Shredder.sln -c Release --no-restore`
- [ ] `dotnet test src/Shredder.Tests/Shredder.Tests.csproj -c Release --no-build`
- [ ] CLI `shredder --help`、`shredder --version` 输出正常。
- [ ] WPF 主界面可启动，拖拽、取消、报告入口可用。

## 安全

- [ ] 系统目录、Program Files、驱动器根目录仍被硬拒绝。
- [ ] Warn 路径仍要求关键字确认，`--yes` 不可绕过 Forbidden。
- [ ] 日志默认不记录原始路径。
- [ ] SSD/NVMe 限制在 README 和 UI 中可见。
- [ ] 回收站和空闲空间擦除均支持取消并清理临时状态。

## 开源发布

- [ ] `LICENSE`、`SECURITY.md`、`CONTRIBUTING.md`、`ROADMAP.md` 存在且内容准确。
- [ ] `.gitignore` 生效，没有 `bin/`、`obj/`、日志、发布包进入提交。
- [ ] `CHANGELOG.md` 顶部对应版本号下条目已写好,日期改成发版日,`[Unreleased]` 留空骨架。
- [ ] Release notes 写清楚破坏性变化、已知限制、升级建议。
- [x] 发布包附带 SHA-256 校验和(已由 `.github/workflows/build.yml` 的 release job 自动生成 `.sha256` 文件并随 zip 上传)。
